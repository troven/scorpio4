Defining Terms

We should probably explain what eg:knows, eg:name and eg:DoB mean. Whilst english speaking humans might understand, non-anglophones and machines will not.

The RDF Schema vocabulary (rdfs:) uses RDF to define a RDF terms. Everything in the Semantic Web is ultimately grounded in the same core rdf: terms.

So, let's introduce our terms. The rdfs:range defines the <object> part of the triple:

        # Our first vocabulary in N3 notation:
        @PREFIX     rdfs:		<http://www.w3.org/2000/01/rdf-schema#> .

        eg:name     rdfs:range          xsd:string.
        eg:DoB      rdfs:range          xsd:date.

Relating Terms

We can define the <subject> part too. Assuming, we mean each term applies to a eg:Person, we would say:

        eg:knows    rdfs:domain         eg:Person.
        eg:DoB      rdfs:domain         eg:Person.
        eg:name     rdfs:domain         eg:Person.

Each statement declares that the <subject> of an eg:knows relationship can be assumed to refer to a eg:Person.

We would probably want to say that eg:Person is the object of the eg:knows relationship too.

        eg:knows    rdfs:range          eg:Person.

The ideas behind RDFS may feel strange to those used to SQL database schemas. These definitions do not actually say that only a eg:Person can only eg:knows a eg:Person.
