Executable Assets

Assets is an Active Vocabulary to manage the lifecycle of executable scripts, queries and templates.

These executable Assets can be triggered automatically or by some event. Of course, that needs they integrate with FLO

They are simply text-based files that are distinguished by their mime type, and fall into three broad categories:

Scripts

Scorpio4 supports any Java Script Engine languages are the runtime environment supports, by default Javascript and Groovy.

How and when each script is run is described by consulting one or more of the Active Vocabularies.

Scripts can be run synchronously or asynchronously, depending upon the execution engine that is interpreting them.

Queries

Queries are instructions to create, read, update or delete data in some data source. Scorpio4 supports SQL and SPARQL queries. Other query engines are planned.

Queries can be expressed in their native form, or using the experimental ASQ language (Abstract Semantic Query) to generate them, particularly useful when inferred from annotated meta-data.

Templates

The rest of the Assets are simply text files that can be used as templates.

Templates can be used to publish generated output back into the knowledge base, or to export it through any API

Two template engines are available, a simple one supporting moustache-like {{xyz}} placeholders or the more powerful and expressive Groovy Template Engine.

  • HTML, XHTML, XML
  • CSV, CSS, TXT
  • Many more ...
