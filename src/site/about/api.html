Scorpio4 API

Scorpio4 constructs a dynamic API that presents itself over a variety of protocols. The most common, of course, is HTTP.

Scorpio4 uses it's semantic model to govern access to your business rules - whether they are expressed in Java, a plethora of (JSR-223) scripting languages or the multitude of integration possibilities that Apache Camel offers.

The API is expressed as a processing pipeline, based on Apache Camel, that routes messages from place to place.

Web API

It is common to expose some part of your application as either an HTTP-based API, web app or site.

The first step is to declare that your application should listen for events from the web-service running on port 9999:

        <urn:Example:>    camel:from  <restlet:http://localhost:9999/hello>;

Secondly, you need to build a processing pipeline that does something with the event:

        <restlet:http://localhost:9999/hello>  camel:to    <urn:Example.script.com.scorpio4.toolchain.exec.hello_world>

The [restlet:] protocol creates a dynamic web service that accept requests on the URI and routes them to the com.scorpio4.toolchain.exec.hello_world script.

Semantic API

Each part of the pipeline is unique URI that can be linked to other meta-data.
