Active Vocabularies

An Active Vocabulary is a vocabulary of terms, usually active verbs that invoke runtime behaviour.

An Active Vocabulary can employ one or more Execution Strategies to activate it's classes and predicates.

An Execution Strategy operates at the intersection of RDF and traditional programming, leveraging existing tools, code and skills.

Scorpio4 programmers focus on concisely expressing their business logic, whilst taking advantage of Scorpio4's dynamic execution environment and contextual meta-data.

Scorpio4 supports three default Execution Strategies:

Executable Assets

Scorpio4's RDF-based Asset Register is the source of truth for your application scripts, queries and templates.

Supported assets include Javascript, Groovy, Python, Ruby, SPARQL, SQL, XML, HTML, CSV and JSON.

An active predicate is an instance of an Executable bean: that can be supported by the runtime.

It is the Executable that determines which mime-types are supported. The predicate identifies a RDF literal with a supported mime-type.

Active Beans

The Active Beans ontology describes a Spring Registry, enabling Scorpio4 to act as an executable source of truth for Spring-based components.

Any Java beans can be declared using the bean: scheme, for example bean:com.example.HelloWorld

The bean: is an rdf:type of bean:Bean

Active FLO

Active FLO is a source of truth for simple, powerful Enterprise Integration Patterns.

The predicates described by the FLO ontology closely mirror Camel's fluent API, with only a few tweaks to support the underlying RDF model.

These Active Predicates are used to model messages passing between two Camel endpoints (from the subject to the object).

It aims to be a faithful but not a perfect abstraction, never-the-less the FLO ontology is powerful enough to encompass all of the Enterprise Integration Patterns.

Active FLO supports Active Beans and Executable Assets are part of a pipeline.
