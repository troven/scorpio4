Design

We take a meta-data driven approach to application development. At the heart of Scorpio4 is an RDF/RDFS data model backed by a dynamic execution environment.

Everything in Scorpio4 is meta-data. We think about your data simply as meta-data about reality, which seems to fits with the RDF/Linked Data/Semantic Web view of the world.

Scorpio4 encourages and empowers you to:

  1. use URIs to identify everything of importance
  2. use URIs as tags to express relationships.
  3. use RDF to formally describe your application domain
  4. use RDFS to give your data a richer "semantic" meaning
  5. use SPARQL rules to generate data models, user interfaces, enforce constraints and apply validation
  6. use SPARQL rules to check for inconsistencies, logic errors and contradictions
  7. use asset templates to generate HTML, CSV, XML, SQL, SPARQL, source code or even RDF.
  8. use Active Vocabularies to describe APIs, build user interfaces, integrations, work flows and business rules.
